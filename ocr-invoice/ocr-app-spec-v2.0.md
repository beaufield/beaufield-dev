## CSV自動変換の仕組み（PowerShell・Windows標準機能）

### 課題
- 販売管理システムが出力する `商品.CSV` は **Shift-JIS形式**
- Claude Codeは **UTF-8** でないと正しく読めない
- 毎日CSVが更新されるため、毎日変換が必要

### 解決策：`csv変換.bat` + Windowsタスクスケジューラ

**csv変換.bat の内容（Windowsに標準搭載のPowerShellを使用）**
```batch
@echo off
powershell -Command ^
  "Get-Content 'D:\Dropbox\ClaudeWork\開発・自動化\ocr-invoice\商品.CSV' -Encoding Default | ^
   Set-Content 'D:\Dropbox\ClaudeWork\開発・自動化\ocr-invoice\商品_utf8.csv' -Encoding UTF8"
echo 変換完了
```

**タスクスケジューラ設定（一度だけ設定）**
- 毎朝 8:00 に `csv変換.bat` を自動実行
- 販売管理システムの出力時刻より後に設定すること
- → 以後は毎日自動で `商品_utf8.csv` が最新状態に更新される

## OCR読み取り仕様

### Claude Codeへの指示プロンプト（チャットで依頼する際の内容）

Claude Codeは以下の情報を納品書PDFから抽出する：

```
抽出する情報：
1. メーカー名（発行元の会社名）
2. 納品日
3. 各商品の明細（商品行のみ・全行分）
   - 商品名
   - JANコード（13桁の数字）
   - メーカー独自コード（JAN以外のコード）
   - 数量
   - 単位
   - 単価
   - 税率区分（8%または10%）

除外する行：
- 値引き行・送料行・割引行・マイナス金額行
- 合計行・小計行
- 数量がない行・商品名がない行

特記事項：
- 回転・横向きPDFも正しく読むこと
- JANコードは13桁の数字列として抽出すること
- 読み取れない項目はnullとすること
```

## 照合ロジック（変わらず）

OCRで読み取った各商品に対して以下の順番で照合する。

### Step 1：JANコードで完全一致（信頼度：高🟢）
- 読み取ったコードが13桁 → `商品_utf8.csv` の「JANCD」列と照合
- 一致 → Beaufieldコード確定（緑表示）

### Step 2：商品名の部分一致（信頼度：中🟡）
- JANコードが不明または一致なし
- `商品_utf8.csv` の「商品名」「かな」列を部分一致検索
- 最大3件の候補を提示

### Step 3：未変換（信頼度：低🔴）
- Step 1・2いずれも該当なし
- 未変換として出力

## 制約・注意事項

- Claude Codeを使うにはインターネット接続が必要
- 1回の処理でPDF複数ページまで対応（ページ数が多い場合は都度確認）
- 商品マスターの列名変更時は Claude Code に再読み込みを依頼すること
- `商品_utf8.csv` は自動上書きされるため編集しないこと（編集は元の`商品.CSV`側で行う）
