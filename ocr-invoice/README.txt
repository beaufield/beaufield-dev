========================================
納品書OCR処理ツール  使い方ガイド
========================================
最終更新：2026年3月27日

----------------------------------------
【このツールでできること】
----------------------------------------
メーカーから届いた納品書PDF を Claude（AI）に読み取らせ、
商品コード・数量・単価・合計が入ったCSVを自動生成します。

出力されたCSVはそのまま弥生会計・販売管理システムへの
取り込みデータとして使えます。

----------------------------------------
【フォルダ構成】
----------------------------------------
ocr-invoice/
├── README.txt              ← このファイル
├── csv変換.bat             ← ① 商品マスター更新に使う
├── 納品書/                 ← ② PDFをここに入れる
│   └── 処理済み/          ←    処理後のPDFが自動移動される
├── results/               ← ③ 出力CSVが保存される
├── 商品.CSV               ←    OneDriveからコピーされる商品マスター（原本）
└── 商品_utf8.csv          ←    Claude が読み込む変換済みマスター

----------------------------------------
【通常の使い方（毎回の手順）】
----------------------------------------

  ▼ STEP 1：商品マスターを更新する
  ─────────────────────────────────
  「csv変換.bat」をダブルクリックする。

  これだけで以下が自動実行されます：
    1. OneDrive（PowerBI\Data\商品.CSV）から最新をコピー
    2. 文字コード変換（CP932 → UTF-8）して 商品_utf8.csv を生成

  「Done! 商品_utf8.csv を更新しました。」と表示されればOK。

  ※ OneDrive の同期が終わっていないとエラーになります。
    タスクバーのOneDriveアイコンが「同期完了」になってから実行してください。


  ▼ STEP 2：納品書PDFを所定フォルダに入れる
  ─────────────────────────────────────────
  処理したい納品書PDFを以下のフォルダに入れる：

    ocr-invoice\納品書\

  ※ 複数ファイルを一度に入れてOKです。
  ※「処理済み」サブフォルダには入れないでください。


  ▼ STEP 3：Claude に依頼する
  ─────────────────────────────
  Claude Code を開いて、以下のひと言を送るだけ：

    「納品書スキャンして」

  Claudeが自動で以下を行います：
    - 商品マスターの鮮度チェック
    - 納品書フォルダのPDFを読み取り
    - 商品コードの照合（JANコード・メーカーコード・商品名）
    - CSVを results フォルダに出力
    - 処理済みPDFを「処理済み」フォルダへ移動


  ▼ STEP 4：CSVを確認する
  ─────────────────────────
  results フォルダに出力されたCSVを開いて確認する。

  ファイル名の形式：
    {メーカー名}_{納品日YYYYMMDD}_{伝票番号}.csv
    例）ナプラ_20260319_0003995405.csv

  コード「1」の行は照合できなかった商品です。
  → 手動で正しいコードに修正してから取り込んでください。

----------------------------------------
【商品が「コード1」になる主なケース】
----------------------------------------
・商品マスターに未登録の商品
・マスターのJANコード・メーカーコードが未設定
・新商品・取扱開始前の商品
・リーフレット・パンフレット類（マスター未登録の場合）

→ 弥生への取り込み前に手修正が必要です。

----------------------------------------
【商品マスターに新商品を追加したいとき】
----------------------------------------
1. PowerBI\Data\商品.CSV（弥生のエクスポートデータ）を更新
2. csv変換.bat を実行して 商品_utf8.csv を再生成
3. 次回から自動で照合されるようになります

----------------------------------------
【トラブルシューティング】
----------------------------------------
▼ csv変換.bat でエラー「ERROR: Source file not found.」
  → OneDrive の同期が完了しているか確認してください。
  → D:\OneDrive - Beaufield\PowerBI\Data\ に商品.CSV があるか確認。

▼ Claudeに「商品マスターが〇日前の状態です」と言われた
  → csv変換.bat を実行してマスターを更新してください。

▼ results フォルダにCSVが出力されない
  → 納品書フォルダにPDFが正しく入っているか確認してください。
  → PDFが「処理済み」フォルダに入っていないか確認してください。

▼ CSVをExcelで開いたら文字化けした
  → ファイルをダブルクリックではなく、
    Excel の「データ」→「テキストファイルから取り込む」で
    UTF-8 を指定して開いてください。

----------------------------------------
【ファイルを誤って消してしまったら】
----------------------------------------
・納品書PDFは「処理済み」フォルダにあります
・CSVは results フォルダにあります
・商品_utf8.csv は csv変換.bat を再実行すれば再生成できます
