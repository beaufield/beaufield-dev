# 発注アプリ 構築指示書 v1.5
# Claude Codeへの指示：この仕様書の通りにアプリを構築してください。

---

## システム構成

```
【フロントエンド】
index.html（GitHub Pages で公開）
↓ スマホのブラウザからURLでアクセス
beaufield.github.io/order-app/

【バックエンド・データ】
Googleスプレッドシート「Beaufield発注管理」
↓ Google Apps Script（GAS）がAPIとして機能
POST/GETリクエストでデータを読み書き

【商品マスター（読み取り専用）】
OneDrive上の商品.CSV（毎日更新）
↓ スマホから手動選択して読み込み
ブラウザのlocalStorageにキャッシュ（商品マスターのみ）
```

---

## 構築条件

- HTML + JavaScript のシングルファイルアプリ（index.html で完結）
- GitHub Pages でホスティング（beaufield.github.io/order-app/）
- ブラウザで動作・インストール不要
- **スマートフォンでの使用をメイン**に設計する（レスポンシブ対応必須）
- 対応ブラウザ：Chrome / Edge
- インターネット接続あり前提（CDN・GAS API利用）
- バーコードリーダー：Bluetooth接続（スマホにキーボード入力として認識される）
- 外部ライブラリはCDN経由でOK

## 重要な実装上の注意事項

### GAS通信方式（必須）
- GASへのPOST通信は **`application/x-www-form-urlencoded` 形式を使用すること**
- JSONでPOSTするとCORSエラーが発生する
- kiki-kanri（beaufield.github.io/kiki-kanri/）のGAS通信コードと同じパターンで実装すること

### PDF生成方式（必須）
- jsPDFの日本語フォント登録は複雑なため使用しない
- **`html2canvas` + `jsPDF` の組み合わせ**で実装する
  1. 印刷用HTMLを非表示divとして画面内に生成する
  2. html2canvasでそのdivを画像として取得する
  3. jsPDFで画像をA4サイズのPDFに変換してダウンロードする
- これにより日本語フォント問題を回避できる

### バーコードリーダーの入力制御（必須）
- Bluetooth接続リーダーは高速なキー入力として届くため、入力確定後に **150msのデバウンス処理**を入れること
- Enterキーの二重送信を防ぐためにフラグ管理を行うこと
- JANコード入力欄には `inputmode="none"` を設定してソフトキーボードを抑制する

### 商品マスターCSVの列特定方式（必須）
- 列番号（何列目）ではなく **ヘッダー行の列名で列を特定**すること
- 販売管理システムの更新で列が増減しても動作が壊れないようにする
- 対象のヘッダー名：「コード」「商品名」「単位名」「仕入先CD」「JANCD」

### 入力データの一時保存（必須）
- 画面B（商品入力中）の入力データを **sessionStorageに随時保存**すること
- ページを再表示・ブラウザが一時停止した場合でも入力内容を復元する
- 発注完了またはキャンセル時にsessionStorageをクリアする

## GAS API エンドポイント

GASをWebアプリとしてデプロイし、以下のAPIを実装する。
通信方式は `application/x-www-form-urlencoded` 形式を使用する。

### GET系（データ取得）

```
?action=getMasters
→ 発注先マスター・担当者マスターを一括取得
→ レスポンス：{ suppliers: [...], staffs: [...] }

?action=getOrders
→ 発注履歴一覧を取得（直近30件）
→ レスポンス：{ orders: [...] }
```

### POST系（データ書き込み）

```
action=saveOrder
→ 発注履歴・発注明細を保存
→ パラメータ：orderNo, date, supplierCode, supplierName, fax,
              staff, outputType, items（JSON文字列）

action=saveSupplier
→ 発注先マスターを追加・更新・削除
→ パラメータ：code, name, fax, mode（add/update/delete）

action=saveStaff
→ 担当者マスターを追加・削除
→ パラメータ：name, mode（add/delete）
```

- 発注担当者（プルダウン）：GASから取得した担当者マスターの名前一覧

## 画面B：商品入力

### JANコードスキャン入力フロー
```
① JANコード入力欄にフォーカス（画面表示と同時に自動フォーカス）
↓
② BluetoothバーコードリーダーでスキャンするとJANコードが自動入力される
  ※リーダーはEnterキー付きで送信・150msデバウンス処理で安定化
↓
③ 数量入力欄に自動フォーカス移動（ソフトキーボードが出る）
↓
④ 数量を入力してEnterキー or 完了ボタン
↓
⑤ 商品マスターCSVからJANコードで商品名・単位を取得してリストに追加
   入力内容をsessionStorageに自動保存
↓
⑥ JANコード入力欄に自動フォーカス戻る（連続スキャン可能）
```

### JANコードが商品マスターに存在しない場合
- 「このJANコードは商品マスターに見つかりません」と警告表示
- 以下の2つの選択肢を提示する
  - **選択肢①：商品名検索モード**（キーワード入力で商品マスターを絞り込む）
  - **選択肢②：手書き追加モード**（商品名を直接手入力する）
- 手書き追加商品には「※」マークを付けて識別（CSV出力対象外）

### 商品名検索モードの動作
```
キーワードを入力（例：「シャンプー」）
↓
商品マスターの商品名・かな列を対象に部分一致検索
↓
候補リストを表示（最大20件）
↓
タップして選択 → 数量入力欄にフォーカス移動
```

### 入力済みリスト（画面下部に常時表示）

| 商品名 | 数量 | 単位 | 備考 | 操作 |
|---|---|---|---|---|
| ○○シャンプー | 10 | 本 | | 削除 |
| ※手書き商品 | 3 | 本 | | 削除 |

- 各行の削除ボタンで個別削除
- 数量・備考は直接編集可能（編集のたびにsessionStorageを更新）
- 1件以上でないと「確認画面へ」ボタンが非活性

## 出力A：PDF生成（FAX注文書）

### 生成方式
- **html2canvas + jsPDF** を使用する（日本語フォント問題を回避）
  1. 非表示divに印刷用HTMLを生成する
  2. html2canvasでdivを画像化する
  3. jsPDFでA4サイズのPDFに変換してダウンロードする
- GAS APIへの発注データ保存と同時に実行する

### ファイル名
「発注先名_YYYYMMDD_発注No.pdf」（例：デミコスメティクス_20260318_001.pdf）

### レイアウト仕様
A4縦サイズで既存のFAX注文書フォーマットを再現する。

```
┌─────────────────────────────────────────┐
│ FAX注文書              beaufield        │
│                        株式会社ビューフィールド │
│ 毎度お世話になっております。              〒889-1605 宮崎市清武町加納甲2323-17
│ 下記の通り注文致しますので、              TEL：0985-85-3031 FAX:0985-85-2814
│ 宜しくご手配の程お願い申し上げます。      （定休日：日曜日・月曜日・祝日）
│
│ ┌────────────────────┐  発注日付 [2026/03/18]
│ │  ○○株式会社         │  御中
│ └────────────────────┘  発注担当 [前島]
│
│  商品名              数量  単位  備考
│ ┌───────────────────────────────────┐
│ │ ○○シャンプー       10   本
│ │ △△カラー剤          5   個   要確認
│ │（残り行は空白）
│ └───────────────────────────────────┘
│
│         FAX [0XX-XXXX-XXXX]  発注No [20260318-001]
└─────────────────────────────────────────┘
```

### 詳細ルール
- 1ページあたり最大14行・15行以上は自動で2ページ目
- 発注Noは「YYYYMMDD-連番3桁」形式で自動採番
- ページ下部のボタン類は含めない

- 商品マスターを読み込んだ後に「対応確認」ボタンで実行する

## 商品マスターCSVの読み込み

### ファイル選択方式
- 「📂 マスター読込」ボタンでファイル選択ダイアログを開く
- OneDriveの商品.CSVを手動で選択して読み込む
- 読み込み後はlocalStorageにキャッシュ（読み込み日時も保存）
- CSVが更新されたときだけ再読込すればよい
- 読み込み完了後に「仕入先CD対応確認」を自動的に促すメッセージを表示する

### 文字コード
Shift-JIS（encoding.js等のライブラリを使用してUTF-8に変換）

### 列の特定方法（重要）
- **列番号ではなくヘッダー行の列名で列を特定すること**
- 1行目をヘッダーとして読み込み、以下の名前で列を特定する

| ヘッダー名 | 用途 |
|---|---|
| 「コード」 | Beaufield商品コード |
| 「商品名」 | 表示用商品名 |
| 「かな」 | 商品名検索用よみがな |
| 「単位名」 | PDF表示用単位 |
| 「仕入先CD」 | 発注先との紐付け |
| 「JANCD」 | スキャン照合キー |

- ヘッダーが見つからない場合はエラーメッセージを表示する

### JANコードが空の商品
- スキャンでは呼び出せない
- 商品名検索モードまたは手書き追加モードで対応する

## UIデザイン方針

- **スマートフォンファースト**で設計する
- タップしやすい大きめのボタン（最小タップ領域：48px以上）
- フォントサイズは大きめ（本文16px以上・入力欄18px以上）
- 画面幅は最大480pxを基準に設計（スマホ縦持ち）
- スクロールは縦方向のみ・横スクロールなし
- キーボードが表示されても操作できるレイアウト
- ボタン間に適切な余白を確保（誤タップ防止）
- Beaufieldのブランドカラー（ゴールド系：#C9A96E）をアクセントに使用
- バージョン番号を画面右下に常時表示する（例：v1.0.0）

## GAS URLの設定

アプリ内にGAS_URLという定数を定義し、GASのWebアプリURLを設定する。
```javascript
const GAS_URL = 'https://script.google.com/macros/s/xxxxx/exec';
const APP_VERSION = 'v1.0.0';
```
GAS_URLはGASをデプロイ後に発行されるURLに差し替える。
