## 🔄 変換仕様

### 入力（OCR CSV）
- ファイル名形式: `{メーカー名}_{YYYYMMDD}_{伝票番号}.csv`
- 文字コード: UTF-8-sig（BOM付き）
- 列: `コード, 商品名, 数量, 単価, 合計`
- 保存先: `ocr-invoice/results/`

### 出力（61列CSV）
- 文字コード: **CP932（Shift-JIS）**
- 形式: `仕入データダウンロード ファイルレイアウト`（日立ソリューションズ）
- 保存先: `output/` フォルダ（ブラウザからダウンロードも可）

### 主要列マッピング

| 列# | 列名 | 値 |
|-----|------|-----|
| 1 | 仕入データID | 自動採番（counter.json管理） |
| 2 | メーカーID | master.jsonのマスターから |
| 3 | ディーラーID | 固定: `A0020800` |
| 4 | 出荷日 | ファイル名から（YYYYMMDD） |
| 5 | 取引区分 | UI選択（11=売上/31=直送） |
| 12 | 伝票番号 | ファイル名から |
| 14 | 取引先名 | 固定: `ｶﾌﾞｼｷｶﾞｲｼｬ ﾋﾞｭｰﾌｨｰﾙﾄﾞ` |
| 44 | 商品コード | JANCD（商品_utf8.csvから照合） |
| 45 | 商品名 | 全角→半角カナ変換・25文字以内 |
| 48 | 発注数量 | OCR数量 |
| 51 | 単価 | OCR単価 |
| 53 | 金額 | OCR合計 |

### 空欄にしている列（設計判断済み）
- お届け先情報（列20〜23）: 空欄でよいとオーナー確認済み
- 入数（列46）: 商品マスターに列なし → 空欄（将来対応可）

## 🏭 メーカーIDマスターについて

**まず最初にやること**: 「マスター管理」タブでメーカーを登録する。

| 設定項目 | 説明 |
|---------|------|
| メーカー名 | OCRファイル名の先頭部分と**完全一致**させること |
| メーカーID | サプライヤーから提供される数値ID |

例: ファイル名が `ナプラ_20260312_12345.csv` なら → メーカー名 = `ナプラ`

メーカーIDの確認方法: `shipment_list_A0020800-SMP.csv`（デスクトップ）の「メーカＩＤ」列

---

## 📊 連番（ID）管理

`counter.json` で管理。変換のたびに自動インクリメント。

```json
{
  "shiire_data_id": 10000001,   ← 伝票ごとに+1
  "shiire_meisai_id": 90000001  ← 行ごとに+1
}
```

**初期値を変更したい場合**: counter.json を直接編集してOK。

---

## 🔧 商品マスター（JANコード）

- 参照先: `../ocr-invoice/商品_utf8.csv`（相対パス）
- 起動時に自動読み込み（再起動で更新反映）
- 商品コード → JANコード を自動マッピング
- 未照合（コード=1）の商品はJAN空欄で出力
