## 権限設計（user_app_rolesシート）

アクセス権限は `beaufield-auth` の `user_app_roles` シートで管理。
`is_admin` カラムとは独立しており、このアプリのみの権限設定。

| role | 対象 | できること |
|------|------|------------|
| `admin` | 事務スタッフ | 全予約閲覧・編集・削除、ステータス変更（確定/予約に戻す）、商品管理、集計、入荷処理 |
| `staff` | 営業スタッフ | 自分の予約のみ登録・変更・削除（「予約」ステータスのみ）、全件閲覧、集計閲覧 |
| 未登録 | — | アクセス不可（ACCESS_DENIED） |

**ユーザー登録方法:** `user_app_roles` シートに `user_id / yoyaku-kanri / admin or staff` を追記。

## 入荷処理機能（v1.8.0〜）

### フロー
1. 予約一覧タブ → 📦入荷処理ボタン（admin限定）
2. Step1: 商品選択（予約ありのもの） + 入荷数を入力
3. Step2: プレビューテーブル（No昇順で自動割り当て）
   - 各行のON/OFFチェック・確定数の手動調整可能
   - リアルタイムカウンター（入荷数 / 割り当て済 / 残り）
   - 超過時はエラー表示・実行ボタン無効化
4. 実行 → 確定フィルタで一覧を表示

### スプリット処理（分割確定）
- 予約数の一部だけ確定する場合：
  - **元行**: 数量を減らす（ステータスは「予約」のまま、Noは変えない → 優先順位保持）
  - **新行**: 確定分を新Noで追加（サロン名・担当・発送方法・予約日時をコピー）

## 速度改善対策（v1.6.0 / v1.10.0）

### GAS側
- **keepWarm()関数**: GASコールドスタート（15〜30秒）を防止
  - **GASエディタ → トリガー → keepWarm → 5分おき** で設定済み
  - `PropertiesService` を1回読むだけの軽量関数。課金・クォータ消費はほぼゼロ
- `CACHE_TTL_AUTH`: 5分 → 15分（認証キャッシュ延長）
- `_getUsersFromAuth`: 10分キャッシュ追加（AUTH_SHEET_ID の二重読み込みを解消）

### フロント側
- **localStorageキャッシュ** (`bf_yoyaku_v1100`, TTL=5分)
  - キャッシュあり → 即表示、バックグラウンドで予約を最新化
  - キャッシュなし → GASから全取得後キャッシュ保存（次回から即表示）
  - 予約の変更操作後はキャッシュも自動更新

## 出荷一覧機能（v1.12.0〜）

### 概要
- admin限定。予約一覧の 🚚 出荷一覧 ボタンからモーダルで表示
- 対象: ステータス「確定」のみ
- フィルタ: 商品（必須）+ 期間（今日/今週/今月/全期間、updated_at基準）
- 集計キー: 担当者 × 発送方法 × サロン名（同一組み合わせを合算）
- ソート: 担当者名 → 発送方法（発送→持参→未定）→ サロン名
- 小計: 発送方法ごと + 担当者ごと + 総合計
- 印刷: A4縦PDF（新ウィンドウ）

## 備考機能（v1.12.0〜）

- reservationsシート N列（14列目）に `notes` を追加
- 予約登録・編集フォームに備考テキストエリアを追加（任意入力）
- 予約カード: 備考入力がある場合のみ小さく表示（📝マーク付き）
- 予約一覧の印刷にも備考列を追加
- 出荷一覧にも備考列を表示
- 入荷処理スプリット時に備考をコピー
- **既存シートへの追加**: N列がないシートでも読み込み時に空文字で返るため後方互換性あり

## 集計タブ 商品フィルタ機能（v1.13.0〜）

- 商品別予約数テーブルの行をタップ → その商品で担当別・サロン別・クロス集計をフィルタ
- 選択行をハイライト表示。再タップで解除
- フィルタ中は「🔍 〇〇〇 でフィルタ中 ✕解除」バナーを表示
- 商品テーブル自体は常に全商品表示（フィルタなし）

---

## 予約一覧 全選択ボタン（v1.14.0〜）

- チェック可能な予約がある場合、リスト上部に「全選択」ボタンを表示
- クリックで全チェック → ボタンが「全解除」に変化
- 再クリックで全解除

## 予約一覧 担当フィルタ連動仕様（v1.14.0〜 / v1.14.1で商品連動強化）

| 操作 | 担当ドロップダウンの内容 |
|------|------------------------|
| 商品フィルタ未選択 | 予約が存在する全担当者 |
| 商品フィルタで商品を選択 | その商品の予約がある担当者のみ |

- 商品を切り替えた際、以前選んでいた担当者がその商品に予約を持たない場合は「すべての担当」に自動リセット
- 予約データ更新のたびに自動再構築（常に最新状態）

## データ構造

### products シート（商品マスター）

| 列 | 内容 | 備考 |
|----|------|------|
| A | product_id | 自動生成（P + timestamp） |
| B | name | 商品名 |
| C | stock_limit | 予約上限数（0=無制限） |
| D | deadline | 受付期限（YYYY-MM-DD） |
| E | is_active | TRUE/FALSE |
| F | created_at | 登録日時 |

### reservations シート（予約データ）

| 列 | 内容 | 備考 |
|----|------|------|
| A | reservation_no | 予約No（最大No+1で発番、欠番あり） |
| B | salon_name | サロン名 |
| C | product_id | 商品ID |
| D | product_name | 商品名（登録時コピー） |
| E | quantity | 数量 |
| F | status | 予約 / 確定 |
| G | staff_id | 担当者 user_id |
| H | staff_name | 担当者名 |
| I | operator_id | 操作者 user_id |
| J | operator_name | 操作者名 |
| K | delivery_method | 発送 / 持参 / 未定 |
| L | reserved_at | 予約日時 |
| M | updated_at | 更新日時 |
| N | notes | 備考（任意・v1.12.0追加） |
